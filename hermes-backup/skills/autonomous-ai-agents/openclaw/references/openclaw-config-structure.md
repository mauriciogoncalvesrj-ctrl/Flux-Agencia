## Adding OpenRouter

1. Add plugin entry:
```json
"openrouter": {
  "enabled": true
}
```

2. Add models under `agents.defaults.models`:
```json
"openrouter/anthropic/claude-sonnet-4": {
  "alias": "Claude Sonnet 4 (OpenRouter)"
},
"openrouter/deepseek/deepseek-chat-v3": {
  "alias": "DeepSeek V3"
}
```

3. Set env var on container: `OPENROUTER_API_KEY`

## Adding Ollama Cloud

1. Add plugin entry:
```json
"ollama": {
  "enabled": true
}
```

2. Add models:
```json
"ollama/qwen3:30b": {
  "alias": "Qwen3 30B"
},
"ollama/llama3.3:70b": {
  "alias": "Llama 3.3 70B"
}
```

3. Set env var on container: `OLLAMA_API_KEY`

## Adding Anthropic (direct)

1. Add plugin entry:
```json
"anthropic": {
  "enabled": true
}
```

2. Add models:
```json
"anthropic/claude-sonnet-4-20250514": {
  "alias": "Claude Sonnet 4"
}
```

3. Set env var on container: `ANTHROPIC_API_KEY`

## Critical Notes

- **Model prefix matters:** The string before the first `/` in the model ID must match a loaded plugin. `openai/gpt-5.4` works because the `openai` plugin is enabled. `openrouter/...` will fail unless the `openrouter` plugin is also enabled.
- **`agents.defaults.model.primary`:** This must match an `alias` from the `models` map. If you remove the aliased model, update this field too.
- **`models.mode: merge`:** This merges plugin-provided models with user-defined ones. Setting it to `replace` would hide the defaults.
- **Restart required:** OpenClaw reads this JSON once at startup. After editing, restart the container.
- **JSON validity:** A malformed config will prevent OpenClaw from starting. Always validate with `python3 -m json.tool` before restarting.
